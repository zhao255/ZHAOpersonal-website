Robot Basketball Competition

Role: Team Leader

Date: First semester 2024

Institution: HK POLYU

Project Overview

Led an 8-member multidisciplinary team in designing and constructing a robot capable of autonomously shooting basketballs into hoops of varying heights. The robot incorporated sensors for spatial awareness, motors for movement and shooting mechanisms, and a sophisticated control system.

Challenges & Solutions

One of our biggest challenges was the robot's shooting mechanism stability. Initially, the robot would tilt when extending its arm to shoot, causing missed shots. After three design iterations, I implemented a counterweight system and modified the center of gravity, ultimately improving accuracy by 65%.

Another significant challenge was sensor integration. The proximity sensors were initially giving inconsistent readings when detecting the hoop's position. I led the team in creating a calibration algorithm that helped normalize sensor readings across different lighting conditions.

Technical Implementation

The robot used an array of technologies:

  • SolidWorks for 3D modeling and simulation testing
  • Python for sensor data processing and control algorithms
  • Arduino microcontrollers for motor control
  • Ultrasonic sensors for distance measurement
  • Computer vision using OpenCV for hoop detection

Project Journey

The competition spanned six months, from initial design to final implementation. We started with brainstorming sessions where we sketched potential robot designs. After selecting our approach, we divided into sub-teams focusing on mechanical design, electronics, and programming.

Weekly progress meetings kept the project on track. When we encountered the stability issue in month three, I organized an intensive two-day workshop where we completely redesigned the shooting mechanism. This pivotal moment transformed our robot from a basic prototype to a competition-ready machine.

The final week before the competition involved round-the-clock testing and fine-tuning. Our efforts paid off when our robot successfully completed 7 out of 8 shooting challenges.

AI-Powered Automation Program Plugin

Type: Personal Research Project

Date: 2025

Project Overview

Developed a Python-based tool that automates repetitive tasks in Excel and across various computer applications. The tool uses machine learning to identify patterns in user behavior and suggests automation opportunities.

Project Genesis

The idea for this project came when I was working on a data analysis assignment that required repetitive formatting and calculation steps in Excel. I realized I was spending more time on mechanical tasks than actual analysis. This sparked my interest in creating a tool that could learn from my actions and eventually automate them.

Technical Implementation

The plugin consists of three main components:

  • A background listener that records user actions in supported applications
  • A pattern recognition engine powered by machine learning algorithms
  • An execution module that can repeat identified patterns with minimal user input

For the machine learning component, I utilized open-source libraries like scikit-learn for pattern recognition and PyAutoGUI for task automation. The most challenging aspect was creating an algorithm that could distinguish between intentional patterns and random actions.

Code Sample

Here's a sample of Python code I've written for automating data processing:

import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

# Load and prepare data
def prepare_data(filename):
    data = pd.read_excel(filename)
    # Clean missing values
    data.fillna(data.mean(), inplace=True)
    
    # Feature engineering
    data['efficiency_ratio'] = data['output'] / data['input']
    
    return data

# Train prediction model
def train_model(data, target_column, test_size=0.3):
    features = data.drop(columns=[target_column])
    target = data[target_column]
    
    X_train, X_test, y_train, y_test = train_test_split(
        features, target, test_size=test_size, random_state=42
    )
    
    model = RandomForestClassifier(n_estimators=100)
    model.fit(X_train, y_train)
    
    # Evaluate model
    predictions = model.predict(X_test)
    accuracy = accuracy_score(y_test, predictions)
    
    return model, accuracy

# Main automation function
def process_mechanical_data(input_file, output_file):
    print("Loading data...")
    data = prepare_data(input_file)
    
    print("Training predictive model...")
    model, accuracy = train_model(data, 'failure_risk')
    print(f"Model accuracy: {accuracy:.2f}")
    
    # Apply predictions to new data
    new_data = data.copy()
    new_data['predicted_risk'] = model.predict(
        new_data.drop(columns=['failure_risk'])
    )
    
    # Save results
    new_data.to_excel(output_file, index=False)
    print(f"Results saved to {output_file}")
    
# Example usage
if __name__ == "__main__":
    process_mechanical_data("mechanical_components.xlsx", "risk_assessment.xlsx")
